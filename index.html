User Experience Design....

User experience design is essential when developing an application and is concerned with how users interact with everyday objects or user interfaces. When creating a good user experience it is important to have a good conceptual model in the design process, to provide a good appearance, instructions, feedback, mapping, visibility and behaviour. The term “UX Design” was first coined by the usability expert Don Norman. Norman’s Book The Design of Everyday Things highlights the seven design principles which are accredited as the core principles of the UX design process.

They are:

  • Conceptual model – To give a better understanding and feeling of control. A sound conceptual model enhances the discoverability and evaluation of the overall design.

  • Mapping – This is concerned with the relationship between the elements of two sets of things, for example, the mapping of the vertical and horizontal scroll bars on an application window corresponds to the direction and positioning of the content in the window.

  • Feedback – Feedback is when an action is committed and that action is confirmed by an instantaneous reaction such as a click of a mouse, the sound signifies the action. Another example would be an onclick event causing some on screen element to animate.

  • Discoverability – Discoverability is when it is possible for the user to determine what actions are achievable in the current state of the application of device. For example, the burger icon easily signifies to the user that there is a dropdown menu, from there the user can discover other functionalities.

  • Signifiers – Used to ensure important aspects of the design clearly visible and legible to the user, this involves using the correct font size, colour scheme and layout. Used effectively signifiers can ensure that the discoverability and feedback can be easily communicated.

  • Constraints – Constraints are concerned with the physical, logical, semantic and cultural constrains to guide the users actions and to ease how the user interprets the design. In Wed development constraints are evident when viewing a page on a mobile device as the layout and interactivity are optimised for ease of use. Constraints limit the number of alternatives.

  • Affordance – Defines what actions are possible. The affordance is the relationship between the properties of a physical object and the users interaction. For example, a chair affords a person to sit, a html button affords the user to hover over and click on to perform a function (Norman, 2013).
